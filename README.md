# Тестовое задание в YADRO (Команда Tatlin.Unified DataPath)

Проект реализует эффективную внешнюю сортировку данных для устройств типа «лента».

---

## Архитектура проекта

Проект построен на принципах **SOLID** и разделен на логические слои:

*   **Интерфейс `ITape`**: Абстракция, описывающая физику ленты (чтение, запись, перемотка, сдвиг). Это позволяет алгоритму работать с любым носителем (файлы, память, сеть).
*   **Класс `FileTape`**: Бинарная реализация ленты. Работает с сырыми данными (`int32_t`), обеспечивая максимальную скорость и честную эмуляцию задержек.
*   **Класс `TapeSorter`**: Ядро проекта. Реализует **сбалансированное двухпутевое слияние (Balanced Two-Way Merge Sort)**. Использует 4 временные ленты для сортировки данных любого объема при минимальном потреблении RAM.
*   **ETL-процесс в `main`**: Внешние интерфейсы удобны для человека (текст), а внутренние процессы оптимизированы для машины (бинарные файлы). Приложение автоматически конвертирует данные перед сортировкой и после неё.


Структура проекта:
* ITape.h — базовый интерфейс.

* FileTape.h / .cpp — бинарная лента (работает с файлами в tmp).

* TapeSorter.h / .cpp — логика сбалансированного слияния.

* TapeConfig.h / .cpp — парсинг ini файла.

* MemoryTape.h — реализация ленты в памяти для тестов (Mock).

* TapeSorterTest.cpp — unit-тесты (GTest).

* main.cpp — конвертация данных и управление жизненным циклом лент.

## Сборка и запуск

Команды для консоли написаны для Linux. Запуск для Windows с аналогичными командами.

### Требования
*   Компилятор с поддержкой **C++20**.
*   **CMake** версии 3.15 или выше.

### Сборка

Находясь в корне проекта:
```bash
mkdir build
cd build
cmake ..
cmake --build .
```

В папке build появятся два исполняемых файла: `tape_sort` (консольное приложение) и `run_tests` (Google Test).

### Запуск

Скопируйте содержимое файла `tape_config.ini.example` в `tape_config.ini`:
```bash
cp file.txt copy_file.txt
```
И подставьте нужные значения (в мс).

Запуск консольного приложения:

```bash
./tape_sort <имя_входного_файла> <имя_выходного_файла> <лимит_памяти_в_байтах>
```
Пример:
```bash
./tape_sort ../tests/in/1.txt ../tests/out/1.txt 4
```

Запуск unit-тестов:
```bash
./run_tests
```
